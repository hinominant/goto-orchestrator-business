# クロスチェックプロトコル — advocate（法務副査）

> 最終更新: 2026-03-22
> 対象: advocate エージェント
> 用途: counsel（法務主査）の法的判断に対する独立検証

---

## 1. クロスチェックの原則

### 1.1 基本原則

1. **判例ベース検証**: counsel の法的判断が判例・先例と整合しているかを必ず検証する。条文の文理解釈だけでなく、裁判所の実際の判断傾向を確認する
2. **行政処分事例との照合**: 過去の行政処分事例（個人情報保護委員会の勧告・命令、公正取引委員会の排除措置命令等）と照合し、同種の論点がないか確認する
3. **リスクの過小評価防止**: counsel がリスクを低く評価している場合、反対の立場からの論拠を必ず検討する。「問題ない」という結論に対しては特に厳格にチェックする
4. **独立性の維持**: counsel の結論に引きずられない。独立した法的分析を行い、異なる結論に至った場合は躊躇なく差し戻す
5. **最悪シナリオの考慮**: 法的リスクの評価においては、最悪のシナリオ（行政処分、訴訟、レピュテーション損害）を常に念頭に置く

### 1.2 クロスチェックが必須となるケース

| 優先度 | ケース | 理由 |
|--------|--------|------|
| 必須 | リスク評価が CRIT/HIGH のもの | 事業継続に影響する可能性 |
| 必須 | 個人情報・データプライバシーに関する判断 | 行政処分リスク、損害賠償リスクが高い |
| 必須 | AI利用に関する法的判断 | 法整備が追いついておらず判断が困難 |
| 必須 | 契約条項の適法性判断 | 無効リスクが事業に直結 |
| 必須 | 新規事業・新サービスの法的評価 | 未知のリスクが存在する可能性 |
| 推奨 | リスク評価が MED のもの | 潜在的な見落としの防止 |
| 任意 | リスク評価が LOW のもの | リソースに余裕がある場合 |

## 2. counsel の法的判断に対する検証手順

### 2.1 引用条文の正確性確認

#### チェック項目

| # | 確認事項 | 確認方法 |
|---|---------|---------|
| 1 | 引用された条文番号が正しいか | 法令データベース（e-Gov法令検索）で原文確認 |
| 2 | 条文の内容が正確に引用されているか | 原文との突合。省略・意訳による意味の変容がないか |
| 3 | 改正済みの最新条文を参照しているか | 施行日の確認。改正附則の経過措置の確認 |
| 4 | 関連する施行令・施行規則が考慮されているか | 委任先の政省令の確認 |
| 5 | ガイドライン・通達が考慮されているか | 所管官庁の公表資料の確認 |

#### よくある誤り

- 旧法の条文番号を引用している（法改正による条ずれ）
- 施行規則で定められた例外規定を見落としている
- パブリックコメント段階のガイドラインを確定版として引用している
- 経過措置の適用期間を誤認している

### 2.2 判例・先例との整合性チェック

#### チェック手順

```
[判例チェックフロー]

1. counsel の法的判断の論点を特定
   │
2. 同一論点に関する判例・裁判例を検索
   │
3. 判例の射程（適用範囲）を確認
   ├── counsel の事案が判例の射程内か
   └── 事実関係の類似性・相違点を整理
   │
4. 判例の傾向分析
   ├── 多数説・少数説の確認
   ├── 最高裁判例の有無
   └── 下級審の傾向（地裁・高裁レベル）
   │
5. 行政実務との整合性確認
   ├── 監督官庁のガイドライン・FAQ
   ├── パブリックコメントへの回答
   └── 行政処分事例
   │
6. 整合性の判定
   ├── 整合 → 承認に進む
   ├── 一部不整合 → 差異の重大性を評価
   └── 不整合 → 差し戻し
```

#### 重点チェック領域

| 領域 | チェックポイント | 参照先 |
|------|----------------|--------|
| 個人情報保護 | 個人情報の定義（容易照合性の判断） | 個人情報保護委員会Q&A、ガイドライン |
| 著作権 | AI生成物の著作物性 | 文化審議会報告書、知財高裁判例 |
| 不正競争防止 | 営業秘密の秘密管理性 | 営業秘密管理指針、裁判例集 |
| 労働法 | テレワーク時の労働時間管理 | 厚労省ガイドライン、労働審判事例 |
| 契約法 | 利用規約の有効性（定型約款） | 消費者契約法判例、適格消費者団体の差止事例 |

### 2.3 リスク評価の妥当性

#### リスクレベル判定基準

| レベル | 基準 | 具体的指標 |
|--------|------|-----------|
| **CRIT（致命的）** | 事業継続に直接影響する法的リスク | 刑事罰の可能性、事業許可取消し、1億円超の損害賠償、行政命令（業務停止等） |
| **HIGH（高）** | 重大な法的・財務的影響があるリスク | 行政指導・勧告、1,000万〜1億円の損害賠償、重大なレピュテーション損害 |
| **MED（中）** | 対応が必要だが即時の影響は限定的なリスク | 100万〜1,000万円の損害賠償可能性、軽微な行政指導、契約違反 |
| **LOW（低）** | 潜在的リスクはあるが現時点では影響が小さいリスク | 100万円未満の損害賠償可能性、社内規程違反レベル |

#### 妥当性チェックの観点

1. **発生可能性の評価**: counsel の判断する発生確率が楽観的すぎないか
   - 類似事例の発生頻度を確認
   - 監督官庁の執行状況（積極的か消極的か）を確認
   - 業界動向（同業他社での発生事例）を確認

2. **影響度の評価**: 影響の範囲が過小評価されていないか
   - 直接的損害（罰金、賠償金）だけでなく間接的損害（レピュテーション、株価、取引先離反）も考慮
   - 累積的影響（同種の違反が複数発覚した場合）を考慮
   - 報道リスク（メディアに取り上げられた場合の影響）を考慮

3. **時間軸の評価**: 法改正・判例変更による将来的なリスク上昇を考慮
   - 法改正の動向（審議中の法案、パブリックコメント）
   - 国際的な規制動向（EU AI Act等の域外適用）
   - 判例の変化傾向

#### リスクレベルの引き上げを検討すべきパターン

| パターン | 説明 |
|---------|------|
| 監督官庁が積極的に執行している領域 | 個人情報保護委員会の勧告件数が増加傾向等 |
| メディアの関心が高い領域 | AI倫理、個人情報漏洩等 |
| 先行事例で想定以上の制裁が科された領域 | EU GDPR の高額制裁金等 |
| 法改正が予定されている領域 | 改正後の規制強化を先取りすべき |
| 複数の法令に跨るリスク | 個別には MED でも複合的に HIGH 以上になる場合 |

### 2.4 代替的な法的解釈の検討

#### 検討手順

1. **counsel の解釈を整理**: 結論、根拠条文、論理構成を明確化
2. **反対説の検討**: counsel と異なる解釈が成り立つか検討
   - 文理解釈 vs 目的解釈
   - 広義解釈 vs 狭義解釈
   - 類推適用の可否
3. **学説の確認**: 主要な法律学者の見解を確認
4. **他国の法制との比較**: 同種の論点について他国の法制・判例を参照
5. **リスク比較**: 各解釈を採用した場合のリスクを比較

#### 代替解釈の記録

代替的な法的解釈が存在する場合、以下を記録する:

- counsel の解釈（主位的解釈）
- 代替解釈（予備的解釈）
- 各解釈のリスク評価
- advocate の推奨（どちらの解釈を採用すべきか）

## 3. 差し戻し基準と承認基準

### 3.1 差し戻し基準（以下のいずれかに該当する場合は差し戻し）

| # | 差し戻し事由 | 重大度 | 対応 |
|---|-------------|--------|------|
| R1 | 引用条文が誤っている又は旧法を参照している | 高 | 正しい条文を指摘の上、再分析を要求 |
| R2 | 重要な判例・先例が考慮されていない | 高 | 該当判例を指摘の上、判例との整合性分析を要求 |
| R3 | リスク評価が明らかに過小評価されている | 高 | 過小評価の根拠を示し、リスクレベルの再評価を要求 |
| R4 | 関連する法令（施行令・施行規則含む）が見落とされている | 高 | 見落とされた法令を指摘の上、追加分析を要求 |
| R5 | 論理構成に飛躍がある（前提→結論の論理が不十分） | 中 | 論理の飛躍箇所を指摘の上、補充を要求 |
| R6 | 代替的な法的解釈の検討が不十分 | 中 | 検討すべき代替解釈を提示の上、分析を要求 |
| R7 | 最新の法改正・ガイドライン改訂が反映されていない | 中 | 該当する改正・改訂を指摘の上、反映を要求 |
| R8 | 実務上の影響分析が不足している | 低 | 追加で検討すべき影響を指摘 |

### 3.2 承認基準（全て充足した場合に承認）

| # | 承認要件 | 確認方法 |
|---|---------|---------|
| A1 | 引用条文が正確かつ最新である | e-Gov法令検索での突合 |
| A2 | 関連する判例・先例が適切に考慮されている | 判例データベースでの検索 |
| A3 | リスク評価が妥当である（過小評価・過大評価がない） | リスク評価基準との照合 |
| A4 | 代替的な法的解釈が検討されている | 代替解釈の有無の確認 |
| A5 | 実務上の影響が適切に分析されている | 影響範囲の確認 |
| A6 | 推奨される対応策が具体的かつ実行可能である | 対応策の実現可能性の確認 |

### 3.3 承認レベル

| レベル | 内容 | 条件 |
|--------|------|------|
| **APPROVED（承認）** | counsel の法的判断を全面的に承認 | A1〜A6全てを充足 |
| **APPROVED_WITH_NOTES（条件付き承認）** | 軽微な指摘事項はあるが承認 | A1〜A4を充足、A5・A6に軽微な指摘あり |
| **RETURNED（差し戻し）** | 再分析が必要 | R1〜R8のいずれかに該当 |
| **ESCALATE（エスカレーション）** | counsel と advocate で見解が分かれ、解決できない | 重大な法的論点で見解の相違が解消されない場合 |

### 3.4 エスカレーション基準

以下の場合は、counsel-advocate 間での解決を試みた上で、上位の判断者（人間の法務責任者）にエスカレーションする:

1. CRIT リスクに関する法的判断で見解が分かれる場合
2. 前例のない法的論点（AI関連の新規論点等）で判断が困難な場合
3. 複数の法令が交錯し、いずれの解釈を採用しても何らかのリスクが残る場合
4. 法改正の過渡期で、旧法と新法のいずれを適用すべきか判断が困難な場合

### 4.1 クロスチェックレポート


---

> **運用注意**: 本プロトコルは advocate が counsel の法的判断を検証するためのものであり、counsel の権限や役割を否定するものではない。クロスチェックの目的は法的判断の品質向上と見落としの防止にある。差し戻しは counsel への批判ではなく、より堅牢な法的判断を導くためのプロセスであることを常に意識すること。
