# Context Hygiene Protocol

> コンテキストウィンドウは有限リソース。計画的に管理し、品質を維持する。

---

## Auto-Compact設定

```bash
CLAUDE_AUTOCOMPACT_PCT_OVERRIDE=80
```

- 80%到達で自動compact発動
- 50%到達時点で手動compact推奨（`/compact`）

---

## コンテキスト管理ルール

### Topic変更時
- 新しいトピックに切り替える場合は `/clear` でコンテキストをリセット
- 前のトピックの成果物は `.agents/PROJECT.md` Activity Log に記録してから `/clear`

### 迷走検知時
- 3回以上同じアプローチを試して失敗 → `/rewind` で最後の成功地点に戻る
- 方針が不明確になったら一度立ち止まり、目的を再確認

### 長チェーン時（Nexus管理）
- 5ステップ以上のチェーン実行時、中間地点でcompact実施
- compact前に以下を `.agents/PROJECT.md` に記録:
  - 完了済みステップとその成果物
  - 残りステップの計画
  - 重要なコンテキスト（判断理由、制約条件）

---

## Compact前チェックリスト

compact実行前に以下を確認:

- [ ] Activity Log に進捗を記録した
- [ ] 重要な判断理由を永続化した
- [ ] 残タスクの計画を明文化した
- [ ] ファイルパスや変数名など再取得コストが高い情報を記録した

---

## アンチパターン

| パターン | 問題 | 対処 |
|----------|------|------|
| コンテキスト肥大化 | 90%超でパフォーマンス劣化 | 50%で手動compact |
| トピック混在 | 無関係な情報がノイズに | `/clear` でリセット |
| 無限リトライ | 同じ失敗を繰り返す | `/rewind` + 方針変更 |
| compact忘れ | 重要情報の喪失 | チェックリスト実施 |
