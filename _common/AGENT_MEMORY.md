# Agent-Scoped Memory Protocol

> エージェントごとのメモリスコープ管理。知識の蓄積と共有を構造化する。

---

## 3スコープモデル

| Scope | Path | 永続性 | 用途 |
|-------|------|--------|------|
| `session` | メモリ内（揮発） | セッション中のみ | 作業中の一時メモ |
| `project` | `.agents/memory/<agent>/` | プロジェクト永続 | プロジェクト固有の学習 |
| `global` | `~/.claude/agent-memory/<agent>/` | グローバル永続 | 全プロジェクト共通の学習 |

---

## アクセス制御

| 操作 | 自スコープ | 他エージェントスコープ |
|------|-----------|---------------------|
| READ | ✓ | ✓（READ ONLY） |
| WRITE | ✓ | ✗ |
| DELETE | ✓ | ✗ |

- 各エージェントは **自分のスコープにのみ** 書き込み可能
- 他エージェントのメモリは **参照のみ** 可能（クロスエージェント参照）

## 書き込みルール

1. **100行/ファイル上限** — 超過時は古いエントリを要約・圧縮
2. **構造化フォーマット** — 自由テキストではなく、日付 + カテゴリ + 内容
3. **重複禁止** — 書き込み前に既存エントリを確認
4. **セッション終了時** — 重要な学習をメモリに永続化

### 書き込みフォーマット

```markdown
## [YYYY-MM-DD] カテゴリ: タイトル

内容（簡潔に）

**学習:** 次回に活かすべきポイント
```

## クロスエージェント参照パターン

| 参照元 | 参照先 | ユースケース |
|--------|--------|-------------|
| Nexus | CEO | 過去の判断パターンを参考にルーティング |
| Builder | Radar | フレーキーテスト履歴を参考に実装 |
| Analyst | CEO | 過去の判断で使用された指標を参照 |
| Radar | Builder | 実装パターンに合わせたテスト設計 |

## _common/MEMORY.md との関係

| レイヤー | プロトコル | 対象 |
|----------|-----------|------|
| ユーザーメモリ | `_common/MEMORY.md` | ユーザーの学習・発見 |
| エージェントメモリ | `_common/AGENT_MEMORY.md`（本書） | エージェント固有の学習 |
| プロジェクト知識 | `.agents/PROJECT.md` | チーム共有知識 |

エージェントメモリはユーザーメモリを補完する。重複しないこと。
