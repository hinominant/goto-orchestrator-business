## 5. AI 出力確認義務の具体的手順

### 5.1 AI 生成コードの確認手順

```
【AI 生成コードのレビューチェックリスト】

□ 1. 機能要件の充足
   - 要求された機能が正しく実装されているか
   - エッジケースが適切に処理されているか

□ 2. セキュリティ
   - ハードコードされた認証情報がないか
   - インジェクション脆弱性がないか
   - 適切な入力バリデーションが実装されているか

□ 3. 個人情報の取扱い
   - 個人情報がログに出力されていないか
   - 個人情報が外部に送信されていないか
   - 適切なアクセス制御が実装されているか

□ 4. ライセンス
   - 使用されているライブラリのライセンスは適切か
   - コピーライト表示の要否を確認したか

□ 5. テスト
   - 自動テストが作成されているか
   - テストカバレッジは十分か
   - 異常系のテストが含まれているか

□ 6. ハルシネーション（幻覚）チェック
   - 存在しない API やライブラリを参照していないか
   - ドキュメントの引用が正確か
   - 法令・規制の引用が正確か（※必ず原典で確認）
```

### 5.2 AI 生成文書の確認手順

```
【AI 生成文書のレビューチェックリスト】

□ 1. 事実関係の正確性
   - 記載された事実・数値が正確か
   - 法令・規制の引用が正確か
   - 日時・バージョン情報が正確か

□ 2. 論理的整合性
   - 内容に矛盾がないか
   - 結論が根拠に基づいているか

□ 3. 機密情報の混入
   - 社外秘情報が含まれていないか
   - 個人情報が含まれていないか

□ 4. バイアス
   - 特定の立場に偏った記述がないか
   - 公平性が確保されているか
```

## 6. 透明性（AI 利用の開示方針）テンプレート

### 6.1 社内向け開示テンプレート

```
【AI ツール利用に関するお知らせ】

当部門では、業務効率化のため以下の AI ツールを導入しています。

1. 利用ツール
   - LM-orchestrator-business（Anthropic Claude API ベース）

2. 利用目的
   - ソフトウェア開発の支援（コード生成、レビュー、テスト作成）
   - ドキュメント作成の支援
   - 設計検討の支援

3. AI の利用範囲
   - AI の出力は必ず人間がレビューし、最終判断は人間が行います
   - 個人情報や機密情報は AI に入力しません
   - AI の出力をそのまま最終成果物とすることはありません

4. データの取扱い
   - AI API へ送信されたデータは、モデルの学習に使用されません
   - 詳細は CROSS_BORDER_TRANSFER.md を参照してください

5. お問い合わせ先
   [担当部署・連絡先]
```

## 7. AI 生成コードの品質責任の整理

### 7.1 責任の所在

| 段階 | 責任者 | 責任内容 |
|------|--------|----------|
| AI への指示 | 利用者 | 適切なプロンプト設計、入力データの管理 |
| AI 出力の生成 | AI 開発者（Anthropic） | モデルの品質・安全性 |
| AI 出力のレビュー | 利用者 | 出力の正確性・適切性の確認 |
| コードの採用判断 | 利用者 | 最終的な採用・不採用の判断 |
| 本番環境へのデプロイ | 利用者/運用チーム | デプロイ判断と品質保証 |
| デプロイ後の問題 | 利用者/運用チーム | AI 出力を採用した以上、最終責任は利用者 |

### 7.2 原則

- **AI が生成したコードであっても、採用した時点で利用者の責任となる**
- AI 出力をレビューせず採用した場合、過失責任が問われる可能性がある
- AI 開発者（Anthropic）の責任は、モデル自体の瑕疵に限定される
