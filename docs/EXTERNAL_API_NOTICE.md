# 外部 API 利用時の法的注意事項

> 本文書は、LM-orchestrator-business が利用・連携する外部 API およびパッケージに関する法的注意事項をまとめたものである。

---

## 1. Anthropic API

### 1.1 利用規約の主要注意点（Acceptable Use Policy）

Anthropic の Acceptable Use Policy（AUP）に基づき、以下の利用は禁止されている。

| カテゴリ | 禁止事項 |
|----------|----------|
| **違法行為** | 違法な活動の促進、犯罪の教唆 |
| **有害コンテンツ** | マルウェア生成、フィッシング、スパム |
| **虚偽情報** | 意図的な偽情報の生成・拡散 |
| **プライバシー侵害** | 個人の監視、ストーキング、同意なきプロファイリング |
| **知的財産侵害** | 著作権侵害を目的としたコンテンツ生成 |
| **差別** | 人種・性別・宗教等に基づく差別的コンテンツ |
| **武器・暴力** | 武器の製造、暴力の促進 |
| **児童保護** | 児童の搾取に関するあらゆるコンテンツ |

**LM-orchestrator-business での対応:**
- ガードレール設定により、AUP 違反となり得る入力を制限
- エージェントのプロンプトに AUP 遵守の指示を組み込み

### 1.2 データ取扱方針

| 項目 | 内容 |
|------|------|
| **API データの学習利用** | 不使用（API 経由のデータはモデル学習に使用しない） |
| **データ保持期間** | 最大30日間（安全性評価・不正利用検知目的） |
| **Zero Data Retention（ZDR）** | エンタープライズプランで利用可能（即時削除） |
| **データの暗号化** | 転送時: TLS 1.2以上、保存時: AES-256 |
| **アクセス制御** | Anthropic 社内でも限定されたスタッフのみアクセス可能 |
| **第三者提供** | 法的要求がない限り、第三者にデータを提供しない |

### 1.3 レート制限と利用上限

| 項目 | 詳細 |
|------|------|
| **リクエストレート制限** | プランに応じた RPM（Requests Per Minute）制限あり |
| **トークン制限** | モデルごとにコンテキストウィンドウの上限あり |
| **月間利用上限** | プランに応じた月間 API 利用量の上限あり |
| **バースト制限** | 短時間の大量リクエストに対する制限あり |

**注意:** レート制限を超過した場合、429 エラーが返される。リトライ処理の実装が必要。

### 1.4 データ保持期間

| データ種別 | 保持期間 | 目的 |
|-----------|---------|------|
| API リクエスト/レスポンス | 最大30日 | 安全性評価、不正利用検知 |
| API メタデータ（利用量等） | 契約期間中 | 課金、利用分析 |
| アカウント情報 | 契約終了後一定期間 | 法的義務の履行 |

**ZDR（Zero Data Retention）オプション利用時:**
- API リクエスト/レスポンスの保持期間: 0日（即時削除）
- メタデータの保持: 通常どおり

## 2. Figma API

### 2.1 Personal Access Token の管理要件

| 要件 | 詳細 |
|------|------|
| **保管方法** | 環境変数（`.env`）に保管。ソースコードにハードコードしない |
| **`.gitignore` への登録** | `.env` ファイルを必ず `.gitignore` に追加 |
| **アクセス範囲** | 必要最小限のスコープを設定 |
| **有効期限** | 定期的にトークンをローテーション（推奨: 90日ごと） |
| **漏洩時の対応** | 即座にトークンを失効させ、新しいトークンを発行 |
| **共有禁止** | トークンは個人に紐づくため、チームメンバー間で共有しない |

### 2.2 API 利用規約の主要制約

| 制約 | 内容 |
|------|------|
| **レート制限** | Figma API にはレート制限がある（具体的な値は公式ドキュメント参照） |
| **商用利用** | API 経由で取得したデータの商用利用には Figma の利用規約に従う |
| **データの再配布** | API 経由で取得した Figma のデータ・デザインを無断で再配布しない |
| **リバースエンジニアリング** | Figma のサービスやプロトコルのリバースエンジニアリングは禁止 |
| **自動化の制限** | 過度な自動リクエストはアカウント停止の対象となる可能性がある |

### 2.3 知的財産に関する注意

- Figma 上のデザインデータの著作権は、原則としてデザインの作成者に帰属する
- API 経由でデザインデータを取得する場合、著作権者の許諾が必要
- LM-orchestrator-business がデザインデータを AI に送信する場合、著作権法上の複製に該当する可能性がある

## 3. npm / npx 経由のパッケージ

### 3.1 ライセンス確認手順

npm パッケージを利用する際は、以下の手順でライセンスを確認する。

#### 手順1: パッケージのライセンスを確認

```bash
# 個別パッケージのライセンス確認
npm info <パッケージ名> license

# インストール済みパッケージのライセンス一覧
npx license-checker --summary
```

#### 手順2: ライセンスの互換性を確認

| ライセンス | 商用利用 | 改変 | 再配布 | 注意点 |
|-----------|---------|------|--------|--------|
| MIT | 可 | 可 | 可 | 著作権表示・ライセンス文の保持が必要 |
| Apache 2.0 | 可 | 可 | 可 | 変更箇所の明示、特許ライセンスの付与 |
| BSD 2-Clause | 可 | 可 | 可 | 著作権表示の保持 |
| BSD 3-Clause | 可 | 可 | 可 | 著作権者名の広告利用禁止 |
| ISC | 可 | 可 | 可 | MIT とほぼ同等 |
| GPL 2.0/3.0 | 可 | 可 | 可 | **コピーレフト: 派生物も GPL で公開必要** |
| LGPL | 可 | 可 | 可 | ライブラリとしてリンクする場合は LGPL 不要 |
| AGPL | 可 | 可 | 可 | **ネットワーク越しの利用でもソース公開必要** |
| Unlicense | 可 | 可 | 可 | パブリックドメイン |

#### 手順3: 禁止・要注意ライセンスの確認

以下のライセンスのパッケージは利用前に法務確認が必要。

- **GPL / AGPL**: コピーレフト条項により、自社コードの公開義務が発生する可能性
- **SSPL**: MongoDB 等で使用。SaaS 提供時に全ソースの公開義務
- **BSL**: 商用利用に制限がある場合がある
- **独自ライセンス**: 個別に内容を確認する必要がある

### 3.2 `npx -y` の法的リスク

`npx -y` は確認なしでパッケージをインストール・実行するコマンドである。

| リスク | 詳細 |
|--------|------|
| **サプライチェーン攻撃** | 悪意のあるパッケージが自動実行される可能性 |
| **ライセンス違反** | ライセンスを確認せずにパッケージを利用するリスク |
| **意図しない依存関係** | 間接的にインストールされるパッケージのリスク |
| **Typosquatting** | パッケージ名の入力ミスにより悪意のあるパッケージが実行されるリスク |

**推奨対応:**

```
1. npx -y は信頼できるパッケージのみに使用する
2. 新しいパッケージを使用する前に、npm のパッケージページで以下を確認:
   - ダウンロード数
   - メンテナンス状況（最終更新日）
   - 既知の脆弱性
   - ライセンス
3. package.json に明示的に追加し、バージョンを固定する
4. npm audit を定期的に実行する
```

## 4. 契約上の注意点

### 4.1 NDA 対象情報の API 送信リスク

| リスク | 詳細 | 対策 |
|--------|------|------|
| **NDA 違反** | NDA 対象の情報を API 経由で外部に送信した場合、秘密保持義務違反となる | `.claudeignore`、`deny` 設定で NDA 対象ファイルを保護 |
| **契約上の損害賠償** | NDA 違反により損害賠償責任が発生する可能性 | 送信前のデータ分類と管理の徹底 |
| **取引先からの信頼喪失** | 秘密情報の漏洩は取引関係の破綻につながる | 従業員教育、利用ルールの整備 |

### 4.2 API 利用に関する契約確認チェックリスト

- [ ] **Anthropic API**
  - 利用規約の最新版を確認したか
  - DPA の内容を確認したか
  - プランに応じた利用制限を把握しているか

- [ ] **Figma API**
  - Personal Access Token の管理体制は整備されているか
  - API 利用規約の制約を把握しているか

- [ ] **npm パッケージ**
  - 使用パッケージのライセンスを確認したか
  - GPL/AGPL 等のコピーレフトライセンスのパッケージが含まれていないか
  - `npm audit` を定期的に実行しているか

- [ ] **NDA・秘密保持契約**
  - NDA 対象情報を AI API に送信しない体制が整備されているか
  - `.claudeignore` と `deny` 設定が適切に構成されているか

- [ ] **業務委託契約**
  - 委託元の契約で AI ツールの利用が禁止されていないか
  - 委託元のデータを AI API に送信することについて承諾を得ているか

### 4.3 契約違反リスクの低減策

1. **データ分類の実施**
   - 全てのプロジェクトデータを機密度に応じて分類する
   - 機密度: 公開可 / 社内限定 / 秘密 / 極秘

2. **技術的保護措置の適用**
   - `.claudeignore`: 秘密以上のファイルを登録
   - `deny`: 極秘ファイルへのアクセスを完全禁止
   - `DATA_PROTECTION_REMINDER`: プロンプトに注意喚起を挿入

3. **定期レビュー**
   - 四半期ごとに API 利用状況と契約遵守状況をレビューする
   - 新しい API の導入時は、法務部門の確認を得る

## 5. 定期確認事項

以下を四半期ごとに確認する。

- [ ] 各 API の利用規約の改訂有無
- [ ] npm パッケージの脆弱性チェック（`npm audit`）
- [ ] ライセンス互換性の再確認
- [ ] API キー・トークンの棚卸しとローテーション
- [ ] NDA 対象情報の保護状況の確認
- [ ] `.claudeignore` と `deny` 設定の棚卸し
