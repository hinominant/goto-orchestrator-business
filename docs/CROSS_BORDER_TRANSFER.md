# 越境移転ガイドライン（個人情報保護法 第28条対応）

> 本文書は、LM-orchestrator-business が Anthropic API（米国）を利用する際の、個人データの越境移転に関する法的要件と実務対応をまとめたものである。

---

## 1. 法的背景

### 1.1 個人情報保護法 第28条の概要

個人情報取扱事業者は、外国にある第三者に個人データを提供する場合、以下のいずれかの要件を満たす必要がある。

1. **十分性認定国への移転**（法第28条第1項括弧書き）
   - 個人情報保護委員会が個人情報保護制度について我が国と同等の水準にあると認めた国
   - 2026年3月時点の認定国: EU/EEA加盟国、英国、韓国
2. **基準適合体制を整備した第三者への提供**（法第28条第1項）
   - 移転先が個人情報の適切な取扱いに関する体制を整備している場合
3. **本人の同意**（法第28条第1項）
   - 本人に対し必要な情報を提供した上で同意を取得

### 1.2 米国の位置づけ

- **米国は十分性認定を受けていない**
- Anthropic は米国法人（サンフランシスコ）であり、APIサーバーは米国に所在する
- したがって、Anthropic API へ個人データを送信する行為は「外国にある第三者への提供」に該当し得る

---

## 2. 基準適合体制の整備

### 2.1 Anthropic API のデータ取扱方針

Anthropic は以下のデータ保護方針を公表している。

| 項目 | 内容 |
|------|------|
| API データの学習利用 | **不使用**（API経由のデータはモデル学習に使用しない） |
| データ保持期間 | API ログは最大30日間保持後に自動削除（安全性評価目的） |
| 暗号化 | 転送時: TLS 1.2以上、保存時: AES-256 |
| データセンター所在地 | 米国（GCP/AWS） |
| SOC 2 Type II | 取得済み |
| データ削除要求 | API利用規約に基づき対応可能 |

### 2.2 基準適合体制として整備すべき事項

個人情報保護法施行規則第16条に基づき、以下を整備する。

- [ ] Anthropic との間で **DPA（Data Processing Agreement）** を締結する（または既存の利用規約・DPA の内容を確認する）
- [ ] 移転先における個人データの取扱いに関する **契約条項** を確認する
- [ ] 移転先の個人情報保護制度について **定期的な確認** を行う体制を構築する
- [ ] 基準適合体制の整備状況について **記録を保存** する

## 3. 本人への情報提供テンプレート

法第28条第2項に基づき、本人の同意を取得する場合は以下の情報を事前に提供しなければならない。

### 3.1 情報提供テンプレート

```
【外国にある第三者への個人データ提供に関するご説明】

1. 移転先の国名
   アメリカ合衆国

2. 当該外国における個人情報保護制度の概要
   - 米国には、連邦レベルでの包括的な個人情報保護法は存在しません
   - カリフォルニア州消費者プライバシー法（CCPA/CPRA）等、州法レベルでの規制が存在します
   - セクター別の規制（HIPAA、COPPA 等）が存在しますが、包括的な監督機関はありません
   - FTC（連邦取引委員会）が不公正・欺瞞的な取引慣行に対する執行を行います
   - 日本の個人情報保護法との主な差異:
     * 包括的な個人情報保護法制がない
     * 独立したデータ保護機関（日本の個人情報保護委員会に相当）が存在しない
     * 本人のアクセス権・訂正権・削除権について連邦法上の統一的な規定がない

3. 移転先が講ずる個人情報の保護のための措置
   - Anthropic社は API 経由で受信したデータをモデルの学習に使用しません
   - データは暗号化された状態で転送・保存されます（TLS 1.2以上/AES-256）
   - API ログは最大30日間保持後に自動削除されます
   - SOC 2 Type II 認証を取得しています
   - データ削除要求に対応します

4. 提供する個人データの項目
   [具体的な項目を記載]

5. 提供の目的
   [AI処理の具体的な目的を記載]

6. お問い合わせ先
   [担当部署・連絡先を記載]
```

## 4. DPA（Data Processing Agreement）チェックリスト

Anthropic との間で確認すべき DPA の主要条項。

### 4.1 必須確認項目

- [ ] **処理の目的と範囲**: API リクエストの処理のみに限定されているか
- [ ] **データの種類と対象者の範囲**: 処理対象のデータカテゴリが明確か
- [ ] **処理期間**: データ保持期間と処理終了後の削除義務が明記されているか
- [ ] **技術的・組織的安全管理措置**: 暗号化、アクセス制御、監査ログ等の措置が規定されているか
- [ ] **再委託（Sub-processor）の制限**: 再委託先の通知義務と承認プロセスがあるか
- [ ] **データ主体の権利対応**: アクセス権・削除権等への協力義務が規定されているか
- [ ] **データ侵害通知**: インシデント発生時の通知期限と内容が規定されているか
- [ ] **監査権**: 処理の適切性を確認するための監査権が留保されているか
- [ ] **契約終了時の措置**: 契約終了後のデータ返却または削除が規定されているか
- [ ] **準拠法と管轄**: 紛争解決の方法が規定されているか

### 4.2 Anthropic 固有の確認事項

- [ ] Anthropic の利用規約（Terms of Service）の DPA 条項を確認したか
- [ ] API データの学習非使用ポリシーが契約上も担保されているか
- [ ] データ保持期間（最大30日）が自組織のポリシーと整合するか
- [ ] Zero Data Retention（ZDR）オプションの利用可否を確認したか

## 5. 実務上の推奨対応

### 5.1 匿名化・仮名加工の活用（最重要）

**最も効果的な対応は、そもそも個人データを API に送信しないことである。**

| 手法 | 説明 | 法的効果 |
|------|------|----------|
| **匿名化** | 特定の個人を識別できないよう加工し、復元不可能な状態にする | 匿名加工情報は個人情報に該当しない → 越境移転規制の対象外 |
| **仮名加工** | 他の情報と照合しない限り特定の個人を識別できないよう加工する | 仮名加工情報は第三者提供が原則禁止（法第42条） |
| **マスキング** | 個人データの一部を不可逆的に置換する | 残余データが個人情報に該当しなければ規制対象外 |

### 5.2 LM-orchestrator-business での技術的対応

```
推奨設定:
1. .claudeignore に個人情報を含むファイルを登録する
2. deny 設定で個人情報ファイルへのアクセスを禁止する
3. DATA_PROTECTION_REMINDER でプロンプト内に注意喚起を挿入する
4. API 送信前のマスキング処理を実装する
```

### 5.3 記録の保存

法第28条第3項（施行規則第18条）に基づき、以下の記録を保存する。

- 基準適合体制の整備状況
- 移転先の個人情報保護制度の定期確認結果
- 本人同意の取得記録（同意方式を採用した場合）
- DPA の締結状況

## 6. GDPR との関係（EU居住者データを扱う場合）

EU居住者の個人データを取り扱う場合、GDPR の規定にも対応する必要がある。

### 6.1 GDPR Article 13/14（情報提供義務）

EU居住者から個人データを取得する際、以下を通知する必要がある。

- データ管理者の身元と連絡先
- DPO（データ保護責任者）の連絡先（選任している場合）
- 処理の目的と法的根拠
- **第三国への移転の有無と、移転の根拠（適切な保護措置）**
- データ保持期間
- データ主体の権利（アクセス権、訂正権、削除権、処理制限権、異議申立権、データポータビリティ権）
- 監督当局への苦情申立権

### 6.2 GDPR Article 22（自動化された意思決定）

- AI による自動化された意思決定（プロファイリングを含む）のみに基づいて、本人に法的効果または同様の重大な影響を及ぼす判断を行うことは原則禁止される
- 例外: 本人の明示的な同意、契約の履行に必要、EU加盟国法による授権
- 該当する場合、本人には以下の権利がある:
  - 人間の介入を要求する権利
  - 自己の見解を表明する権利
  - 決定に対して異議を述べる権利

### 6.3 GDPR 越境移転規制（Chapter V）

- 米国への移転には **EU-US Data Privacy Framework** の適用可否を確認する
- Anthropic が DPF に参加していない場合、**標準契約条項（SCC）** の締結が必要
- 移転影響評価（TIA: Transfer Impact Assessment）の実施を検討する
