# AI経由個人情報漏洩インシデント対応フロー（個人情報保護法 第26条対応）

> 本文書は、AI（Anthropic API 等）を利用する過程で個人情報の漏えい等が発生した場合の対応フローを定めるものである。

---

## 1. AI コンテキストへの個人情報誤入力は「漏えい」に該当するか

### 1.1 判断フロー

```
個人情報が AI の API に送信された
    │
    ├─ API プロバイダ（Anthropic）は「第三者」に該当するか？
    │   ├─ YES → 本人の同意なき第三者提供（法第27条違反の可能性）
    │   └─ 委託先として整理可能か？
    │       ├─ YES（DPA締結済み）→ 委託先への提供（法第27条第5項第1号）
    │       │   └─ 委託先の監督義務（法第25条）の履行状況を確認
    │       └─ NO → 第三者提供に該当
    │
    ├─ 送信されたデータは「個人データ」に該当するか？
    │   ├─ 特定の個人を識別できる情報を含む → YES
    │   ├─ 他の情報と容易に照合して個人を識別できる → YES
    │   └─ 統計情報・匿名加工情報のみ → NO（漏えいに該当しない）
    │
    └─ 「漏えい」の定義に該当するか？
        ├─ 漏えい: 個人データが外部の第三者に知られる状態に置かれること
        ├─ API 送信 → Anthropic のサーバーで処理される → 外部に送信されている
        └─ Anthropic が学習に使用しない場合でも、
           Anthropic の管理下にデータが一時的に存在する
           → 漏えいに該当し得る
```

### 1.2 判断基準

| 状況 | 漏えい該当性 | 理由 |
|------|-------------|------|
| API に個人データを送信（DPA なし） | **該当する可能性が高い** | 同意なき第三者提供 + 外部送信 |
| API に個人データを送信（DPA 締結済み） | **委託として整理可能** | ただし、意図しない送信は委託の範囲外の可能性 |
| .claudeignore で保護されたファイルの内容が漏出 | **該当する** | 技術的保護措置の不備による漏えい |
| 匿名化済みデータを送信 | **該当しない** | 個人データに該当しない |

## 2. 報告対象4類型との照合チェックリスト

個人情報保護法施行規則第7条に基づく、個人情報保護委員会への報告が義務付けられる4類型。

### チェックリスト

- [ ] **類型1: 要配慮個人情報の漏えい等**
  - 病歴、犯罪歴、障害、人種、信条等の要配慮個人情報が含まれていたか
  - → 1件でも該当すれば報告義務あり

- [ ] **類型2: 財産的被害のおそれがある漏えい等**
  - クレジットカード番号、銀行口座情報等が含まれていたか
  - 不正利用により財産的被害が生じるおそれがあるか
  - → 1件でも該当すれば報告義務あり

- [ ] **類型3: 不正の目的をもって行われたおそれがある漏えい等**
  - 不正アクセス、内部犯行等による漏えいか
  - サイバー攻撃、マルウェア等が原因か
  - → 1件でも該当すれば報告義務あり

- [ ] **類型4: 1,000人を超える漏えい等**
  - 漏えい等の対象となる本人の数が1,000人を超えるか
  - → 超える場合は報告義務あり

## 3. 速報テンプレート（発覚から3〜5日以内）

個人情報保護委員会への速報は、事態を知った日から **3〜5日以内** に行う。

```
【個人データの漏えい等に関する報告（速報）】

報告日: YYYY年MM月DD日
報告者:
  事業者名: [事業者名]
  担当部署: [部署名]
  担当者名: [氏名]
  連絡先:   [電話番号・メールアドレス]

1. 概要
   発生日時: YYYY年MM月DD日 HH:MM頃
   発覚日時: YYYY年MM月DD日 HH:MM頃
   事案の概要:
     [例] AI開発ツール（Claude Code / LM-orchestrator-business）の利用過程で、
     個人情報を含むファイルが意図せず Anthropic API に送信された。

2. 漏えい等が発生した個人データの項目
   [例] 氏名、メールアドレス、電話番号
   ※ 現時点で判明している範囲

3. 漏えい等が発生した個人データの件数
   [例] 約XX件（調査中）

4. 原因
   [例] .claudeignore への登録漏れにより、個人情報を含むファイルが
   AI のコンテキストに含まれ、API リクエストとして送信された。

5. 二次被害の有無・おそれ
   [例] Anthropic の API データ学習非使用ポリシーにより、
   データがモデル学習に使用されるリスクは低い。
   ただし、API ログとして最大30日間保持される可能性がある。

6. 該当する報告対象類型
   □ 要配慮個人情報の漏えい等
   □ 財産的被害のおそれがある漏えい等
   □ 不正の目的をもって行われたおそれがある漏えい等
   □ 1,000人を超える漏えい等

7. 本人への通知状況
   □ 通知済み（YYYY年MM月DD日）
   □ 通知予定（YYYY年MM月DD日予定）
   □ 未通知（理由: ）

8. 再発防止策（暫定）
   [例] 該当ファイルを .claudeignore に追加、deny 設定を強化

9. 確報の提出予定日
   YYYY年MM月DD日（発覚日から30日以内）
```

```
2. 漏えい等が発生した個人データの項目（確定版）
   [確定した項目一覧]

3. 漏えい等が発生した個人データの件数（確定版）
   [確定した件数]

   (1) 直接原因:
       [例] 個人情報を含むCSVファイルがプロジェクトルート直下に配置されており、
       .claudeignore に未登録であった
   (2) 間接原因:
       [例] 個人情報ファイルの配置ルールが未整備であった
   (3) 根本原因:
       [例] 個人情報の取扱いに関する運用規程が策定されていなかった

5. 二次被害の状況（確定版）
   [二次被害の有無と詳細]

6. 本人への通知状況（確定版）
   通知方法: [メール/書面/電話]
   通知日: YYYY年MM月DD日
   通知内容: [概要]

7. Anthropic へのデータ削除依頼の状況
   依頼日: YYYY年MM月DD日
   依頼方法: [メール/サポートポータル]
   対応状況: [削除完了/対応中/未対応]

8. 再発防止策（恒久対応）
   (1) 技術的対策:
       - .claudeignore の見直しと網羅的な登録
       - deny 設定による個人情報ファイルへのアクセス禁止
       - DATA_PROTECTION_REMINDER の設定
       - API 送信前のマスキング処理の導入
   (2) 組織的対策:
       - 個人情報取扱規程の策定
       - 従業員への教育・研修の実施
       - 定期的な設定レビューの実施
   (3) 実施スケジュール:
       [各対策の実施予定日]

9. 添付資料
   - 調査報告書
   - 是正措置の実施記録
```

## 5. Anthropic へのデータ削除依頼手順

### 5.1 手順

1. **Anthropic のサポートに連絡**
   - メール: support@anthropic.com（または利用規約で指定された連絡先）
   - Anthropic Console 内のサポートポータル

2. **削除依頼に含める情報**
   ```
   件名: Data Deletion Request - API Usage Data

   Organization ID: [組織ID]
   API Key (last 4 digits): [下4桁]
   Date/Time range of affected requests: [UTC]
     From: YYYY-MM-DD HH:MM UTC
     To:   YYYY-MM-DD HH:MM UTC
   Description of data to be deleted:
     [送信された個人データの概要（個人データそのものは含めない）]
   Reason for deletion request:
     Accidental transmission of personal data subject to
     Japan's Act on the Protection of Personal Information (APPI).
   ```

3. **対応確認**
   - Anthropic からの削除完了通知を受領する
   - 削除完了の記録を保存する

### 5.2 注意事項

- Anthropic の API データ保持期間は最大30日間であるため、30日経過後は自動削除される
- ただし、法令上の義務として削除依頼を行い、記録を残すことが重要
- Zero Data Retention（ZDR）オプションを契約している場合、データは即座に削除される

## 7. 予防措置チェックリスト

### 7.1 技術的予防措置

- [ ] **`.claudeignore` の設定**
  - 個人情報を含む可能性のあるファイル・ディレクトリを登録
  - 例: `*.csv`, `*.xlsx`, `data/personal/`, `customers/`

- [ ] **`deny` 設定**
  - CLAUDE.md の `deny` ディレクティブで個人情報ファイルへのアクセスを禁止
  - 例: `deny: data/customers/**`

- [ ] **`DATA_PROTECTION_REMINDER` の設定**
  - プロンプトに個人情報取扱いの注意喚起を挿入
  - 例: 「個人情報を含むファイルを読み込まないでください」

- [ ] **ディレクトリ構造の整備**
  - 個人情報を含むデータは専用ディレクトリに集約
  - 当該ディレクトリを `.claudeignore` と `deny` の対象に設定

- [ ] **API 送信前のフィルタリング**
  - 個人情報パターン（メールアドレス、電話番号等）の検出と自動マスキング

### 7.2 組織的予防措置

- [ ] **個人情報取扱規程** の策定
- [ ] **AI ツール利用ガイドライン** の策定と周知
- [ ] **従業員教育** の実施（AI利用時の個人情報リスク）
- [ ] **定期レビュー** の実施（.claudeignore、deny 設定の棚卸し）
- [ ] **インシデント対応訓練** の実施
